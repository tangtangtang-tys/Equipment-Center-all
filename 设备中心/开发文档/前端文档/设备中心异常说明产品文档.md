# 设备中心异常说明产品文档

> 钉钉节点：Obva6QBXJw9wK69as2xqEMPOWn4qY5Pr

## 一、文档目标

本文档用于整理设备中心当前排障页面的产品能力缺口。 
建议后续页面不要只展示原始信息，而是围绕“用户反馈的问题”给出明确结论、证据和下一步操作。

核心目标：
1. 让客服或运维进入设备详情后，第一眼知道设备当前有哪些异常。
2. 每类异常都能看到“为什么判断异常”。
3. 页面能提示当前还缺少哪些信息。
4. 页面能给出下一步排查或操作建议。

## 二、页面总览建议

设备详情页顶部建议增加“设备健康概览”，统一展示各模块状态。

| 模块 | 状态示例 | 作用 |
|------|------------|------|
| 基础状态 | 正常 / 未激活 / 被禁用 / 未知 | 判断设备能不能被正常使用 |
| 服务大区 | 正常 / 疑似异常 / 未知 | 判断数据是否可能查错区域 |
| 用户绑定 | 正常 / 未绑定 / 绑定异常 / 多用户 | 判断用户有没有权限看到设备 |
| 实时连接 | 在线 / 离线 / 连接异常 / 未知 | 判断用户能不能实时预览 |
| 云存储/增值服务 | 正常 / 未开通 / 已过期 / 未生效 | 判断购买服务是否真的生效 |
| 报警日志 | 正常 / 无报警 / 有报警无文件 / 异常 | 判断设备是否产生事件和录像 |
| 低功耗状态 | 正常 / 心跳异常 / 电量低 / 未知 | 判断低功耗设备是否正常醒来和上传 |
| 物联网卡 | 正常 / 缺失 / 白名单异常 / 套餐异常 | 判断 4G 设备联网是否正常 |

## 四、设备基础状态异常

### 问题说明

基础状态用于判断设备是否具备正常使用的前提。如果设备未激活、被禁用、型号异常或固件异常，后续连接、报警、云存储都可能异常。

### 常见异常

| 异常 | 用户侧表现 | 页面应说明 |
|------|---------------|---------------|
| 设备未激活 | App 无法正常使用 | 设备尚未首次上线 |
| 设备被禁用 | 用户无法绑定或使用 | 设备当前不可用，需要确认禁用原因 |
| 型号异常 | 页面无法判断能力 | 设备型号缺失或不支持当前功能 |
| 固件异常 | 连接、云存储、低功耗表现异常 | 固件版本可能过旧或无法识别 |

### 当前页面缺少什么

| 当前缺口 | 建议补齐 |
|------------|------------|
| 基础信息分散 | 顶部增加“基础状态”卡片 |
| 无自动判断 | 页面自动给出正常/异常/未知 |
| 无处理建议 | 对未激活、禁用、型号异常分别给出建议 |

## 五、服务大区异常

### 问题说明

服务大区决定设备详情、日志、云存储、实时连接等数据从哪个区域查询。大区异常会导致“设备能查到，但日志、服务或连接状态查不到”。

### 常见表现

| 表现 | 说明 |
|------|------|
| 设备详情正常，但报警日志为空 | 可能日志在其他区域 |
| 云存储或增值服务查不到 | 服务数据可能在其他区域 |
| 实时连接状态异常 | 连接状态可能查错区域 |
| 修改大区后数据变化 | 原大区可能不准确 |

### 当前页面缺少什么

| 当前缺口 | 建议补齐 |
|------------|------------|
| 不知道哪些数据受影响 | 展示大区可能影响的模块 |
| 修改大区风险不明显 | 增加操作风险和二次确认 |

## 六、用户绑定异常

### 问题说明

用户绑定关系决定用户能不能在 App 看到设备、控制设备，以及购买的服务是否作用在正确用户和设备上。

### 常见异常

| 异常 | 用户侧表现 | 页面应判断 |
|------|---------------|---------------|
| 用户未绑定设备 | App 看不到设备 | 反馈用户不在绑定列表中 |
| 用户绑定错设备 | 用户说买了服务但当前设备没有 | 服务可能开到其他设备 |
| 设备绑定错用户 | 设备属于 A，但显示绑定 B | 当前设备归属异常 |
| 绑定关系已解除 | 以前能看到，现在看不到 | 是否有解绑记录 |
| 多用户权限异常 | 多人能看到但权限混乱 | 主用户、共享用户权限是否清晰 |
| 用户账号混淆 | 用户说绑定了但查不到 | 是否登录了另一个账号 |
| 组织归属异常 | 经销商或组织看不到设备 | 组织层级是否匹配 |
| 服务归属不一致 | 用户买了云存储但没生效 | 服务开通用户与绑定用户是否一致 |

### 当前页面缺少什么

| 当前缺口 | 建议补齐 |
|------------|------------|
| 绑定信息藏在 Tab 中 | 顶部增加用户绑定摘要 |
| 无反馈用户核对 | 支持输入用户信息核对是否绑定 |
| 主用户、共享用户不够直观 | 增加角色标签 |
| 服务归属没有联动 | 对比绑定用户和服务开通用户 |
| 缺少历史状态 | 展示解绑时间、操作来源、增加解绑、绑定日志 |

## 七、实时连接异常

### 问题说明

实时连接用于判断用户能不能在 App 中实时预览、对讲、远程控制设备。

### 常见异常

| 异常 | 用户侧表现 | 页面应说明 |
|------|---------------|---------------|
| 设备离线 | 预览打不开 | 设备没有近期上线记录 |
| 实时连接异常 | 设备在线但画面打不开 | 设备在线，但实时链路异常 |
| 连接身份缺失 | App 无法建立连接 | 设备连接身份数据异常 |
| 连接身份被禁用 | 实时预览失败 | 当前连接身份不可用 |
| 新旧连接链路不兼容 | 部分设备连接失败 | 可能需要评估切换链路 |
| 低功耗设备特殊 | 看起来离线 | 可能处于正常休眠 |

### 当前页面缺少什么

| 当前缺口 | 建议补齐 |
|------------|------------|
| 新旧连接模式展示不一致 | 两种模式都展示统一字段 |
| 没有综合结论 | 自动判断“离线”还是“在线但连接异常” |
| 缺少最近登录信息 | 展示最近登录时间、网络信息、版本信息 |
| 高风险操作提示不足 | 切换连接链路、解绑连接身份前提示影响 |
| 低功耗设备容易误判 | 明确标识低功耗休眠逻辑 |

## 八、云存储与增值服务异常

### 问题说明

用户购买云存储或增值服务后，服务需要真正作用到当前设备。只看到订单存在，不代表服务已经生效。

### 生效链路
```text
用户购买服务
  → 服务绑定到正确设备
  → 设备在线
  → 设备具备联网能力
  → 服务配置下发成功
  → 设备产生报警
  → 文件上传成功
  → 用户才能看到录像或服务效果

```

### 常见异常

| 异常 | 用户侧表现 | 页面应判断 |
|------|---------------|---------------|
| 没有订单 | 用户说买了但查不到 | 是否真的购买，是否买到其他账号 |
| 订单绑定错设备 | 当前设备无服务 | 订单设备是否匹配 |
| 服务已过期 | 回放不可用 | 服务是否仍在有效期 |
| 服务未启用 | 买了但没效果 | 功能开关是否打开 |
| 设备离线 | 服务无法下发 | 设备是否在线 |
| 4G 联网异常 | 4G 设备无法上传 | 卡、白名单、套餐是否正常 |
| 有报警无文件 | 有事件但无录像 | 上传链路或服务配置异常 |
| 设备未触发事件 | 没有任何录像 | 是否有报警事件 |
| 影子系统未下发 |  | 增加查看影子的按钮 |
| app监控网关版本回复的问题 |  | 增加跳转客户端监控网关连接 |

| 漏查点 | 说明 |
|---------|------|
| 影子不是所有服务都有 | D019、D020、D024、D026、H001、AI101 明确写影子；其他服务多依赖主表、缓存或事件中心 |
| PMS 下发失败不会阻断开通 | D025、D026、D027 中 `viot&#95;param.open(...).catch(() =&gt; console.log('下发失败'))`，订单可能成功但设备配置未生效 |
| Redis 参数影响 PMS 下发 | PMS 下发参数来自 Redis DB 37 的 `vip&#95;params:{code}`，模板缺失会回退 `vip&#95;params:default`，可能导致参数不符合服务 |
| VIP 判断有 Redis 缓存 | `vipDeviceCache` 使用 Redis DB 6，缓存未刷新时可能出现服务已变更但 VIP 状态仍旧 |
| 4G 云存储使用独立表 | D021/D022/D023 写 `4G&#95;cloudService`，不是普通 `cloudService` 表 |
| AI 服务依赖授权系统 | AI001-AI008 除了主表，还必须有 `license-service` 授权成功 |
| AI100 不写影子 | AI100 通过运营配置 `custom-config-admin` 同步，不是影子链路 |
| 节点缓存刷新是异步的 | `刷新节点缓存` 会并发刷 1-4 大区，失败写 `log&#95;error&#95;refresh`，不一定阻断订单开通 |

## 九、报警日志异常

### 问题说明

报警日志用于判断设备是否真的产生事件，以及事件是否形成录像、图片或其他资源文件。

### 常见异常

| 异常 | 用户侧表现 | 页面应判断 |
|------|---------------|---------------|
| 无报警 | 用户说有动作但页面无记录 | 设备是否在线、配置是否开启、是否查错日期 |
| 有摘要无明细 | 有数量但列表为空 | 日期、分页或大区是否异常 |
| 有报警无文件 | 有事件但无录像 | 上传链路、云存储、低功耗是否异常 |
| 动作类型不符合预期 | 用户说人形，但日志是其他事件 | 动作识别或映射是否异常 |
| 文件不可访问 | 有文件但打不开 | 文件是否过期或链接异常 |
| 时间查错 | 用户说今天，页面查不到 | UTC 和北京时间是否跨天 |
| 查错大区 | 当前区域无日志 | 是否存在跨区数据 |

### 当前页面缺少什么

| 当前缺口 | 建议补齐 |
|------------|------------|
| 报警摘要未可视化 | 展示近 30 天趋势 |
| 只显示动作文字 | 同时展示原始动作信息 |
| 元数据不易读 | 分组展示关键字段 |
| 文件只下载第一条 | 改成文件抽屉，展示文件列表 |
| 无文件时没有原因 | 展示“有报警无文件”的诊断提示 |
| 不展示查询大区 | 显示本次查询区域 |
| 不支持跨区查询 | 增加跨区对比 |
| 不支持北京时间查询 | 增加北京时间/UTC 切换 |
| 不联动云存储和低功耗 | 增加相关状态摘要 |

### 页面建议展示
```text
报警日志：有报警无文件

所选日期：2026-06-09
报警数量：12
资源文件：无
查询大区：中国大区
云存储：已开通

初步判断：
设备已产生报警，但未生成录像或图片文件，建议继续检查上传链路、云存储配置和低功耗上传状态。

```

## 十、低功耗状态异常

### 问题说明

低功耗设备可能长期休眠，不一定随时在线。排查时应关注设备有没有醒来、有没有报警、有没有上传、有没有生成文件。

### 常见异常

| 异常 | 用户侧表现 | 页面应判断 |
|------|---------------|---------------|
| 设备未唤醒 | 没有报警、没有录像 | 是否因低电量、网络或策略导致未醒 |
| 已唤醒未报警 | 设备有动作但无报警 | 报警配置或动作识别是否异常 |
| 已报警未上传 | 有报警但无录像 | 上传链路是否异常 |
| 已上传无文件 | 有上传但无文件 | 文件服务或回调是否异常 |
| 电量低 | 设备不触发、不上传 | 当前电量是否过低 |
| 过快休眠 | 报警后未完成上传 | 休眠策略是否影响上传 |
| 时间跨天 | 用户反馈时间查不到 | 是否跨时区或跨天 |

### 当前页面缺少什么

| 当前缺口 | 建议补齐 |
|------------|------------|
| 低功耗入口弱 | 在顶部和报警日志中都展示低功耗状态 |
| 只看日志列表 | 改成低功耗诊断时间线 |
| 没有报警联动 | 按某次报警前后查看唤醒、上传、文件 |
| 没有风险标签 | 自动标识低电量、未上传、过快休眠 |
| 没有云存储联动 | 展示服务和配置是否支持上传 |

### 页面建议展示
```text
低功耗状态：异常

当前电量：18%
最近唤醒：10:01:12
最近报警：10:01:20
最近上传：无

诊断结论：
设备已唤醒且产生报警，但没有发现上传记录，建议检查上传链路和设备休眠策略。

```

## 十一、物联网卡异常

### 问题说明

物联网卡用于判断 4G 设备是否具备联网和上传能力。卡异常会导致设备离线、报警无法上传、云存储不生效。

### 常见异常

| 异常 | 用户侧表现 | 页面应判断 |
|------|---------------|---------------|
| 没有卡信息 | 设备离线或无法上传 | 是否缺卡或卡信息未同步 |
| 未加入白名单 | 换卡后不能用 | 设备是否被限制 |
| 机卡绑定异常 | 4G 不上线 | 设备和卡是否匹配 |
| 卡类型未知 | 无法判断卡归属 | 是否第三方卡或卡数据缺失 |
| 插卡位置异常 | 内置/外置判断不一致 | 页面来源是否矛盾 |
| 套餐异常 | 4G 离线、上传失败 | 套餐是否生效、过期或欠费 |
| 卡状态未刷新 | 页面状态不准 | 是否需要刷新卡状态 |

### 当前页面缺少什么

| 当前缺口 | 建议补齐 |
|------------|------------|
| 只展示插卡记录 | 升级为 4G 联网诊断 |
| 没有状态摘要 | 展示卡号、白名单、卡类型、套餐状态 |
| 表格字段少 | 增加是否当前卡、来源、最近出现时间 |
| 没有风险提示 | 自动提示白名单、套餐、卡类型风险 |
| 没有卡套餐入口 | 支持查询套餐是否生效 |
| 没有联动云存储/报警 | 4G 设备无文件时联动卡状态 |

### 页面建议展示
```text
物联网卡：白名单异常

当前卡号：89860************
机卡绑定：支持
白名单：未加入
卡类型：物联网卡
套餐状态：未知

初步判断：
当前设备支持机卡绑定，但未加入白名单，可能影响 4G 联网和报警上传。

```

## 十二、4G 流量卡套餐异常

### 问题说明

用户购买 4G 流量卡套餐后，设备仍然离线、报警无法上传或云存储不生效，需要判断套餐是否真的生效在当前设备使用的卡上。

### 常见异常

| 异常 | 用户侧表现 | 页面应判断 |
|------|---------------|---------------|
| 当前设备没有卡号 | 无法判断套餐 | 设备是否有卡或卡信息未同步 |
| 查不到套餐 | 用户说买了但页面无记录 | 订单是否同步，订单和卡号是否匹配 |
| 套餐未生效 | 买了但暂时不能用 | 是否未到生效时间 |
| 套餐已过期 | 设备离线或上传失败 | 套餐是否过期 |
| 订单卡号不匹配 | 买到其他卡 | 订单对应卡号是否为当前设备卡号 |
| 当前卡不在插卡记录中 | 换卡后异常 | 设备是否真的使用该卡 |

### 当前页面缺少什么

| 当前缺口 | 建议补齐 |
|------------|------------|
| 无套餐查询入口 | 在物联网卡模块增加套餐查询 |
| 无生效状态判断 | 展示未生效、生效中、已过期、未查到 |
| 无订单和卡号匹配判断 | 显示订单卡号是否等于当前设备卡号 |
| 无 4G 流量卡和 4G 云存储区分 | 页面明确两者不是同一类服务 |
| 无卡流量/停机等状态 | 后续补齐卡状态和剩余流量 |
