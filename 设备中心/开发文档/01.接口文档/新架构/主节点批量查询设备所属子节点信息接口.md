# 主节点批量查询设备所属子节点信息接口

> 钉钉节点：m9bN7RYPWdlGLXlMfkq4bOyLWZd1wyK0

## 1\. 接口概述

本接口用于其他业务系统根据批量设备ID查询设备的所属子节点信息接口。

## 2\. 基本信息
- **请求方式**: POST
- **请求URL**: `/api/v1/devices/batch-query-attribution`
- **协议**: HTTP
- **数据格式**: JSON

测试环境地址：[http://192.168.252.124:28628](http://192.168.252.124:28628)

## 3\. 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| deviceIds | array | 是 | 设备唯一标识符vuid,如\['DEV001', 'DEV002', 'DEV003'\] |

## 4\. 请求头

| 头部字段 | 必填 | 说明 |
|------------|------|------|
| Accept | 否 | 指定响应格式，建议使用 `application/json` |
| X-Request-Id | 是 | 请求唯一标识，使用<span style="color: rgb(15, 17, 21);">UUID</span> |

## 5\. 响应参数

### 5.1 基础响应结构

| 字段 | 类型 | 说明 |
|------|------|------|
| code | integer | 响应状态码，200表示成功 |
| message | string | 响应消息描述 |
| data | array\[object\] | 设备数据对象，成功时返回 |
| successCount | int | 查询成功个数 |
| failCount | int | 查询失败个数 |
| failDeviceIds | array | 查询失败的设备ID列表 |

### 6.2 data 对象结构

| 字段 | 类型 | 说明 |
|------|------|------|
| did | string | 设备id |
| cluster | string | 子节点编码 |
| updateTime | timesptap | 子节点更新时间，毫秒 |
| regionId | int | 子节点城市编码 |

## 7\. 响应示例

### 7.1 成功响应 (HTTP 200)
```json
{
  "code": 200,
  "message": "success",
  "data": [{
    "did": "VPEG1000015OQMQ", 
    "cluster": "dev1", 
    "updateTime": 1772675895236, 
    "regionId": 5 
  }],
  "successCount":1, //查询成功个数
  "failCount":1, //查询失败个数
  "failDeviceIds":["VN0017299VPZJ"] //查询失败的设备ID列表
}
```

## 8\. 错误码说明

| 错误码 | HTTP状态码 | 说明 |
|---------|-------------|------|
| 200 | 200 | 请求成功 |
| 400 | 400 | 请求参数错误 |
| 429 | 429 | 请求过于频繁，已被限流 |
| 500 | 500 | 服务器内部错误 |

## 9\. 使用限制
- **请求频率限制**: 单次http查询请求，最大1000台，最低1台。相同IP每分钟最多请求60次.
