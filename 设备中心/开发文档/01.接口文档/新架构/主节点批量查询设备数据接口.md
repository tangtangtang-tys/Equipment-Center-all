# 主节点批量查询设备数据接口

> 钉钉节点：20eMKjyp81RkjOR3i47bqq9yWxAZB1Gv

## 1\. 接口概述

本接口用于其他业务系统根据批量设备ID查询设备的详细信息、状态及实时数据。该接口提供设备基础信息、运行状态和最新采集数据的查询功能，适用于设备监控、状态分析等场景。

## 2\. 基本信息
- **请求方式**: POST
- **请求URL**: `/api/v1/devices/batch`
- **协议**: HTTP
- **数据格式**: JSON

测试环境地址：[http://192.168.252.124:28628](http://192.168.252.124:28628)

## 3\. 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| deviceIds | array | 是 | 设备唯一标识符vuid,如\['DEV001', 'DEV002', 'DEV003'\] |

## 4\. 请求头

| 头部字段 | 必填 | 说明 |
|------------|------|------|
| Accept | 否 | 指定响应格式，建议使用 `application/json` |
| X-Request-Id | 是 | 请求唯一标识，使用<span style="color: rgb(15, 17, 21);">UUID</span> |

## 5\. 响应参数

### 5.1 基础响应结构

| 字段 | 类型 | 说明 |
|------|------|------|
| code | integer | 响应状态码，200表示成功 |
| message | string | 响应消息描述 |
| data | array\[object\] | 设备数据对象，成功时返回 |
| successCount | int | 查询成功个数 |
| failCount | int | 查询失败个数 |
| failDeviceIds | array | 查询失败的设备ID列表 |

### 6.2 data 对象结构

| 字段 | 类型 | 说明 |
|------|------|------|
| did | string | 设备id |
| ver | string | 固件版本号 |
| model | string | 固件标识 |
| ip | string | 设备真实ip |
| node | string | 设备连接进程节点唯一标识,设备ip\+进程ID,如192.168.252.11@123456 |
| type | int | 连接类型  <br>1：ipc mqtt 2:车载tcp  3.BK tcp  4.NVR mqtt 5.宠物 mqtt  6.新TCP  7.老的tcp（低功耗系统）  |
| status | int | 设备状态，0 离线 1 在线 2 休眠 |
| l\_at | long | last\_active\_time简写，最后活动时间。设备最后一次上传数据、更新属性或响应命令的时间戳（毫秒） |
| l\_conn | long | last\_connect\_time简写，最后连接时间（毫秒） |
| l\_disconn | long | last\_disconnect\_time简写，最后断开连接时间（毫秒） |
| battery | int | 当前电量,百分比，低功耗设备才有 |
| iccid | long | 设备sim卡id,4G设备才有 |
| imei | long | <span style="color: rgb(15, 17, 21);">国际移动设备识别码</span> |
| tf | int | 是否有tf卡  0 没有  1有 |
| latitude | double | 纬度,车载才有 |
| longitude | double | 经度,车载才有 |
| speed | double | 速度km/h,车载才有 |

## 8\. 错误码说明

| 错误码 | HTTP状态码 | 说明 |
|---------|-------------|------|
| 200 | 200 | 请求成功 |
| 400 | 400 | 请求参数错误 |
| 429 | 429 | 请求过于频繁，已被限流 |
| 500 | 500 | 服务器内部错误 |

## 9\. 使用限制
- **请求频率限制**: 单次http查询请求，最大1000台，最低1台。相同IP每分钟最多请求60次.
