# B2B Page Specification Checklist

Use this checklist before building or reviewing an admin page.

## Purpose And Roles

- Page goal
- User roles
- Entry path
- Return path
- Primary business object
- Success metric or business outcome

## Information Architecture

- Main regions
- Primary and secondary actions
- Navigation relationship
- Tab or section rules
- Important summaries or counters

## Filters And URL State

- Filter fields
- Default filter values
- Required filters
- Reset behavior
- URL query state
- Shareable URL behavior
- Date/time timezone rules

## Table

- Table fields
- Column labels
- Default sorting
- Fixed columns
- Hidden or configurable columns
- Pagination rule
- Row selection rule
- Row status display
- Row operation list
- Empty table behavior

## Batch Operations

- Batch operation list
- Selection constraints
- Confirmation rules
- Partial success strategy
- Retry strategy
- Audit requirements

## Forms

- Form fields
- Required fields
- Field validation
- Field dependencies
- Default values
- Disabled fields
- Readonly fields
- Submission behavior
- Draft or unsaved warning behavior

## State Machine And Actions

- Business states
- Allowed transitions
- Forbidden transitions
- Available actions per state
- Dangerous actions
- Undo or rollback rules

## API Contract

- Request parameters
- Response fields
- Pagination format
- Error format
- Permission error format
- Idempotency key rules
- Concurrency or version rules

## Page States

- Loading state
- Empty state
- Error state
- Forbidden state
- Partial success state
- Offline or timeout behavior when relevant

## Tracking And Audit

- Analytics events
- Audit log fields
- Operator identity
- Before and after values
- Export or download traceability

## Acceptance Criteria

- Functional criteria
- Interaction criteria
- Permission criteria
- Data safety criteria
- Performance criteria
- Accessibility criteria
- Test criteria
